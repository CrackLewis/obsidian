## 问题现象

树莓派 5 通过网线直连 Windows 电脑（适配器名 `RaspNet`，IP `192.168.137.1`），但无法被发现和连接。

## 诊断过程

### 第一步：检查适配器状态

```powershell
Get-NetAdapter -Name "RaspNet"
```

结果正常：**链路已启用，速度 1 Gbps**。

### 第二步：检查 IP 配置

```powershell
Get-NetIPAddress -InterfaceAlias "RaspNet"
```

IP 为 `192.168.137.1/24`，路由表正常。但：

```powershell
Get-NetConnectionProfile -InterfaceAlias "RaspNet"
```

关键指标：**IPv4Connectivity = NoTraffic**——无 IPv4 流量通过。

### 第三步：检查数据包统计

```powershell
Get-NetAdapterStatistics -Name "RaspNet"
```

| 指标 | 值 | 含义 |
|------|-----|------|
| ReceivedBytes | > 0 | Pi 在发数据 |
| ReceivedUnicastPackets | 0 | Pi 没收到任何单播回复 |
| ReceivedBroadcastPackets | > 0 | Pi 在发广播包 |

**结论：树莓派在发 DHCP Discover 广播请求 IP，但 Windows 没有 DHCP 服务器响应。**

### 第四步：检查 SD 卡 boot 分区

SD 卡的 bootfs 分区（D:）上有 `network-config` 文件，内容为 netplan 格式。该文件在**首次启动时**由 cloud-init 读取并应用，首次启动后不再生效。

- `network-config` 中只配了 WiFi（HUAWEI_B320_49F1），没有 eth0 配置。
- eth0 默认走 DHCP，但直连电脑没有 DHCP 服务器，故拿不到 IP。

## 解决方案

### 方案一（当前使用）：Windows 临时 DHCP 服务器

由于无法直接登录树莓派（无 IP），也无法通过 WSL 挂载 ext4 分区（USB 读卡器不兼容 WSL2 的 Hyper-V 直通），最简方案是在 Windows 上运行一个临时 DHCP 服务器，给 Pi 分配一个 IP。

**原理：** 使用双 Socket 方式：
- `UdpClient(67)` 绑定到 `0.0.0.0:67` 接收 DHCP Discover
- 另一个 `UdpClient` 绑定到 `192.168.137.1:0` 专门在 RaspNet 接口上发送 OFFER/ACK

**执行时机：** 树莓派开机等待 60 秒后运行。

**DHCP 服务器脚本** (`dhcp_server.ps1`)：

```powershell
$ErrorActionPreference = "Stop"

$serverIp = "192.168.137.1"
$offerIp = "192.168.137.2"

$offerAddr = [System.Net.IPAddress]::Parse($offerIp)
$subnetAddr = [System.Net.IPAddress]::Parse("255.255.255.0")
$dnsAddr = [System.Net.IPAddress]::Parse("8.8.8.8")
$serverId = [System.Net.IPAddress]::Parse($serverIp)
$leaseTime = 86400

# 接收器：监听 0.0.0.0:67（接收 DHCP Discover）
$receiver = New-Object System.Net.Sockets.UdpClient(67)
$receiver.EnableBroadcast = $true
$receiver.Client.ReceiveTimeout = 10000

# 发送器：绑定到指定接口（192.168.137.1）
$sendEp = New-Object System.Net.IPEndPoint ([System.Net.IPAddress]::Parse($serverIp), 0)
$sender = New-Object System.Net.Sockets.UdpClient
$sender.Client.Bind($sendEp)
$sender.EnableBroadcast = $true

$startTime = Get-Date

try {
    while ($true) {
        $elapsed = [int](Get-Date).Subtract($startTime).TotalSeconds
        if ($elapsed -gt 120) { break }

        try {
            $remoteEp = New-Object System.Net.IPEndPoint ([System.Net.IPAddress]::Any, 0)
            $data = $receiver.Receive([ref]$remoteEp)

            if ($data.Length -lt 240) { continue }
            $mc = $data[236..239]
            if ($mc[0] -ne 99 -or $mc[1] -ne 130 -or $mc[2] -ne 83 -or $mc[3] -ne 99) { continue }

            $clientMac = $data[28..33]
            $xid = $data[4..7]

            $msgType = 0
            $i = 240
            while ($i -lt $data.Length) {
                if ($data[$i] -eq 255) { break }
                if ($data[$i] -eq 0) { $i++; continue }
                $optLen = $data[$i+1]
                if ($data[$i] -eq 53 -and $optLen -eq 1) { $msgType = $data[$i+2]; break }
                $i += 2 + $optLen
            }

            if ($msgType -eq 1) {
                # DHCP DISCOVER → 发送 OFFER
                $reply = New-Object byte[] 274
                $reply[0]=2; $reply[1]=1; $reply[2]=6; $reply[3]=0
                $xid.CopyTo($reply, 4); $reply[10]=0x80
                $offerAddr.GetAddressBytes().CopyTo($reply, 16)
                $serverId.GetAddressBytes().CopyTo($reply, 20)
                $clientMac.CopyTo($reply, 28)
                $reply[236]=99; $reply[237]=130; $reply[238]=83; $reply[239]=99

                $idx=240
                $reply[$idx]=53; $reply[$idx+1]=1; $reply[$idx+2]=2; $idx+=3
                $reply[$idx]=54; $reply[$idx+1]=4
                $serverId.GetAddressBytes().CopyTo($reply, $idx+2); $idx+=6
                $reply[$idx]=51; $reply[$idx+1]=4
                $lb=[System.BitConverter]::GetBytes($leaseTime)
                if ([System.BitConverter]::IsLittleEndian) { [Array]::Reverse($lb) }
                $lb.CopyTo($reply, $idx+2); $idx+=6
                $reply[$idx]=1; $reply[$idx+1]=4
                $subnetAddr.GetAddressBytes().CopyTo($reply, $idx+2); $idx+=6
                $reply[$idx]=3; $reply[$idx+1]=4
                $serverId.GetAddressBytes().CopyTo($reply, $idx+2); $idx+=6
                $reply[$idx]=6; $reply[$idx+1]=4
                $dnsAddr.GetAddressBytes().CopyTo($reply, $idx+2); $idx+=6
                $reply[$idx]=255

                $bcastEp = New-Object System.Net.IPEndPoint ([System.Net.IPAddress]::Broadcast, 68)
                $sender.Send($reply, $reply.Length, $bcastEp)
            }
            elseif ($msgType -eq 3) {
                # DHCP REQUEST → 发送 ACK（完成）
                $reply = New-Object byte[] 274
                $reply[0]=2; $reply[1]=1; $reply[2]=6; $reply[3]=0
                $xid.CopyTo($reply, 4); $reply[10]=0x80
                $offerAddr.GetAddressBytes().CopyTo($reply, 16)
                $serverId.GetAddressBytes().CopyTo($reply, 20)
                $clientMac.CopyTo($reply, 28)
                $reply[236]=99; $reply[237]=130; $reply[238]=83; $reply[239]=99
                $idx=240
                $reply[$idx]=53; $reply[$idx+1]=1; $reply[$idx+2]=5; $idx+=3
                $reply[$idx]=54; $reply[$idx+1]=4
                $serverId.GetAddressBytes().CopyTo($reply, $idx+2); $idx+=6
                $reply[$idx]=51; $reply[$idx+1]=4
                $lb=[System.BitConverter]::GetBytes($leaseTime)
                if ([System.BitConverter]::IsLittleEndian) { [Array]::Reverse($lb) }
                $lb.CopyTo($reply, $idx+2); $idx+=6
                $reply[$idx]=1; $reply[$idx+1]=4
                $subnetAddr.GetAddressBytes().CopyTo($reply, $idx+2); $idx+=6
                $reply[$idx]=3; $reply[$idx+1]=4
                $serverId.GetAddressBytes().CopyTo($reply, $idx+2); $idx+=6
                $reply[$idx]=6; $reply[$idx+1]=4
                $dnsAddr.GetAddressBytes().CopyTo($reply, $idx+2); $idx+=6
                $reply[$idx]=255

                $bcastEp = New-Object System.Net.IPEndPoint ([System.Net.IPAddress]::Broadcast, 68)
                $sender.Send($reply, $reply.Length, $bcastEp)
                break  # IP 分配完成
            }
        }
        catch [System.Net.Sockets.SocketException] { }
    }
}
finally {
    $receiver.Close()
    $sender.Close()
}
```

### 方案二（推荐）：进入树莓派后固化静态 IP

SSH 登录后（密码为烧录系统时设的密码）：

```bash
ssh cracklewis@192.168.137.2
```

然后执行以下命令设置静态 IP：

```bash
sudo tee /etc/netplan/99-static.yaml << 'EOF'
network:
  version: 2
  renderer: networkd
  ethernets:
    eth0:
      dhcp4: false
      addresses:
        - 192.168.137.2/24
      routes:
        - to: default
          via: 192.168.137.1
      nameservers:
        addresses:
          - 8.8.8.8
          - 8.8.4.4
EOF

sudo netplan apply
```

验证：
```bash
ip a show eth0
ping 192.168.137.1
```

设置后，树莓派重启也会自动用 `192.168.137.2` 启动，随时可以 SSH。

## 根因总结

| 问题 | 原因 |
|------|------|
| Pi 无 IP | eth0 默认 DHCP，但直连电脑无 DHCP 服务器 |
| network-config 不生效 | 仅首次启动由 cloud-init 处理，之后不会再读取 |
| WSL 无法挂载 ext4 | WSL2 的 Hyper-V HCS 层不支持 USB 读卡器直通 |
| OFFER 无法送达 | `UdpClient(67)` 发送广播时可能走错网卡，需单独绑定接口 IP 发送 |
