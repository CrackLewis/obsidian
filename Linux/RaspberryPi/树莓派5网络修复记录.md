- eth0 默认走 DHCP，直连电脑没有 DHCP 服务器，故拿不到 IP

## 解决方案

### 阶段一：Windows 临时 DHCP 服务器

由于无法直接登录树莓派（无 IP），也无法挂载 ext4 分区，最简方案是在 Windows 上运行一个临时 DHCP 服务器。

**核心技术难点：**
- `UdpClient(67)` 绑定到 `0.0.0.0:67` 能接收 DHCP Discover，但发送 OFFER 时会从**默认路由接口**发出（WiFi），不走 RaspNet，导致树莓派收不到
- 解决方案：双 Socket 架构——一个接收器 + 一个绑定到 `192.168.137.1` 的发送器

**DHCP 服务器脚本（`dhcp_server.ps1`）：**

```powershell
$ErrorActionPreference = "Stop"

$serverIp = "192.168.137.1"
$offerIp = "192.168.137.2"

$offerAddr = [System.Net.IPAddress]::Parse($offerIp)
$subnetAddr = [System.Net.IPAddress]::Parse("255.255.255.0")
$dnsAddr = [System.Net.IPAddress]::Parse("8.8.8.8")
$serverId = [System.Net.IPAddress]::Parse($serverIp)
$leaseTime = 86400

# 接收器：监听 0.0.0.0:67（接收 DHCP Discover）
$receiver = New-Object System.Net.Sockets.UdpClient(67)
$receiver.EnableBroadcast = $true
$receiver.Client.ReceiveTimeout = 10000

# 发送器：绑定到指定接口（192.168.137.1），确保 OFFER 从 RaspNet 发出
$sendEp = New-Object System.Net.IPEndPoint ([System.Net.IPAddress]::Parse($serverIp), 0)
$sender = New-Object System.Net.Sockets.UdpClient
$sender.Client.Bind($sendEp)
$sender.EnableBroadcast = $true

$startTime = Get-Date

try {
    while ((Get-Date).Subtract($startTime).TotalSeconds -le 120) {
        try {
            $remoteEp = New-Object System.Net.IPEndPoint ([System.Net.IPAddress]::Any, 0)
            $data = $receiver.Receive([ref]$remoteEp)

            if ($data.Length -lt 240) { continue }

            # 验证 DHCP Magic Cookie
            $mc = $data[236..239]
            if ($mc[0] -ne 99 -or $mc[1] -ne 130 -or $mc[2] -ne 83 -or $mc[3] -ne 99) { continue }

            $clientMac = $data[28..33]
            $xid = $data[4..7]

            # 解析 DHCP 消息类型
            $msgType = 0
            $i = 240
            while ($i -lt $data.Length) {
                if ($data[$i] -eq 255) { break }
                if ($data[$i] -eq 0) { $i++; continue }
                $optLen = $data[$i+1]
                if ($data[$i] -eq 53 -and $optLen -eq 1) {
                    $msgType = $data[$i+2]
                    break
                }
                $i += 2 + $optLen
            }

            if ($msgType -eq 1) {          # DHCP DISCOVER
                $reply = [byte[]]::new(274)
                $reply[0]=2; $reply[1]=1; $reply[2]=6; $reply[3]=0
                $xid.CopyTo($reply, 4); $reply[10]=0x80
                $offerAddr.GetAddressBytes().CopyTo($reply, 16)
                $serverId.GetAddressBytes().CopyTo($reply, 20)
                $clientMac.CopyTo($reply, 28)
                $reply[236]=99; $reply[237]=130; $reply[238]=83; $reply[239]=99

                $idx=240
                $reply[$idx]=53; $reply[$idx+1]=1; $reply[$idx+2]=2; $idx+=3
                $reply[$idx]=54; $reply[$idx+1]=4
                $serverId.GetAddressBytes().CopyTo($reply, $idx+2); $idx+=6
                $reply[$idx]=51; $reply[$idx+1]=4
                $lb=[System.BitConverter]::GetBytes($leaseTime)
                if ([System.BitConverter]::IsLittleEndian) { [Array]::Reverse($lb) }
                $lb.CopyTo($reply, $idx+2); $idx+=6
                $reply[$idx]=1; $reply[$idx+1]=4
                $subnetAddr.GetAddressBytes().CopyTo($reply, $idx+2); $idx+=6
                $reply[$idx]=3; $reply[$idx+1]=4
                $serverId.GetAddressBytes().CopyTo($reply, $idx+2); $idx+=6
                $reply[$idx]=6; $reply[$idx+1]=4
                $dnsAddr.GetAddressBytes().CopyTo($reply, $idx+2); $idx+=6
                $reply[$idx]=255

                $bcastEp = New-Object System.Net.IPEndPoint ([System.Net.IPAddress]::Broadcast, 68)
                $sender.Send($reply, $reply.Length, $bcastEp)
            }
            elseif ($msgType -eq 3) {      # DHCP REQUEST
                $reply = [byte[]]::new(274)
                $reply[0]=2; $reply[1]=1; $reply[2]=6; $reply[3]=0
                $xid.CopyTo($reply, 4); $reply[10]=0x80
                $offerAddr.GetAddressBytes().CopyTo($reply, 16)
                $serverId.GetAddressBytes().CopyTo($reply, 20)
                $clientMac.CopyTo($reply, 28)
                $reply[236]=99; $reply[237]=130; $reply[238]=83; $reply[239]=99
                $idx=240
                $reply[$idx]=53; $reply[$idx+1]=1; $reply[$idx+2]=5; $idx+=3
                $reply[$idx]=54; $reply[$idx+1]=4
                $serverId.GetAddressBytes().CopyTo($reply, $idx+2); $idx+=6
                $reply[$idx]=51; $reply[$idx+1]=4
                $lb=[System.BitConverter]::GetBytes($leaseTime)
                if ([System.BitConverter]::IsLittleEndian) { [Array]::Reverse($lb) }
                $lb.CopyTo($reply, $idx+2); $idx+=6
                $reply[$idx]=1; $reply[$idx+1]=4
                $subnetAddr.GetAddressBytes().CopyTo($reply, $idx+2); $idx+=6
                $reply[$idx]=3; $reply[$idx+1]=4
                $serverId.GetAddressBytes().CopyTo($reply, $idx+2); $idx+=6
                $reply[$idx]=6; $reply[$idx+1]=4
                $dnsAddr.GetAddressBytes().CopyTo($reply, $idx+2); $idx+=6
                $reply[$idx]=255

                $bcastEp = New-Object System.Net.IPEndPoint ([System.Net.IPAddress]::Broadcast, 68)
                $sender.Send($reply, $reply.Length, $bcastEp)
                break
            }
        } catch [System.Net.Sockets.SocketException] { }
    }
} finally {
    $receiver.Close()
    $sender.Close()
}
```

**运行结果：** DHCP DISCOVER → OFFER → REQUEST → ACK 完整流程通过，树莓派获得 `192.168.137.2`。Ping 成功（TTL=64），SSH 服务响应。

### 阶段二：固化静态 IP

首次 DHCP 分配后 SSH 登录，发现网络栈存在冲突：

- **NetworkManager**（`active`）和 **systemd-networkd**（`active`）同时运行
- 此前写入的 `/etc/netplan/99-static.yaml`（`renderer: networkd`）未生效
- eth0 实际由 NetworkManager 管理（`Wired connection 1`）

**改用 NetworkManager 直接配置：**

```bash
nmcli con mod "Wired connection 1" \
  ipv4.method manual \
  ipv4.addresses 192.168.137.2/24 \
  ipv4.gateway 192.168.137.1 \
  ipv4.dns "8.8.8.8 8.8.4.4"

nmcli con down "Wired connection 1" && nmcli con up "Wired connection 1"
```

**最终网络状态验证：**

| 检查项        | 结果                                                        |
| ---------- | --------------------------------------------------------- |
| IP 地址      | `192.168.137.2/24 scope global noprefixroute`（静态，无租约时间）   |
| NM 模式      | `ipv4.method: manual`                                     |
| 路由表        | `default via 192.168.137.1 dev eth0 proto static`（单条，无重复） |
| DHCP 客户端进程 | 无（dhcpcd inactive/not enabled）                            |
| DNS        | 8.8.8.8 / 8.8.4.4（NetworkManager 管理）                      |
| 开机自启       | `autoconnect: yes`                                        |

### 阶段三：配置互联网共享（NAT）

树莓派通过 Windows 的 WiFi 上网：

```powershell
# 启用 IP 转发
Set-NetIPInterface -InterfaceAlias "WLAN" -Forwarding Enabled
Set-NetIPInterface -InterfaceAlias "RaspNet" -Forwarding Enabled

# 创建 NAT
New-NetNat -Name "RaspNat" -InternalIPInterfaceAddressPrefix 192.168.137.0/24

# 防火墙规则
New-NetFirewallRule -DisplayName "NAT RaspPi Forward" -Direction Outbound -InterfaceAlias "WLAN" -Action Allow -Profile Public
New-NetFirewallRule -DisplayName "NAT RaspPi Inbound" -Direction Inbound -InterfaceAlias "RaspNet" -Action Allow -Profile Public
```

验证：`ping 8.8.8.8` 成功（延迟 ~185ms，经 WiFi 路由）。

## 根因总结

| 问题 | 原因 |
|------|------|
| 树莓派无 IP | eth0 默认 DHCP，但直连电脑无 DHCP 服务器 |
| network-config 不生效 | 仅首次启动由 cloud-init 处理，后续启动不读取 |
| WSL 无法挂载 ext4 | WSL2 HCS 层不支持 USB 读卡器直通 |
| DHCP OFFER 无法送达 | `UdpClient(67)` 广播从默认路由接口（WiFi）发出，不走 RaspNet，需双 Socket 方案 |
| netplan 静态 IP 未生效 | NetworkManager 和 systemd-networkd 同时运行，eth0 实际由 NetworkManager 管理 |
| 断电断连 | DHCP 租约到期后无续约服务 |
