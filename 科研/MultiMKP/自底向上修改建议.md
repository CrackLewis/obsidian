当前标题：

```latex

```

略显宽泛。建议改为：

文中多处使用：

```text
maximum plex
multi-k maximum plexes
```

这在英语论文中不够标准。建议统一为：

```text
maximum \(k\)-plex

```

例如原句：


建议改为：

```latex
returns a set of solutions \(\{P_k \mid 2 \le k \le K\}\), where each \(P_k\) is a maximum \(k\)-plex of \(G\).
```

这样可以避免 “plex” 单独出现造成术语不清。

## 3. 第一段应突出“为什么 bottom-up 有效”

当前开头主要是在描述算法顺序：

```text
solves the maximum 2-plex first, then the 3-plex, and so forth
```

但论文写作中更重要的是说明这种顺序的动机。建议补充如下逻辑：

1. 任意 (k)-plex 也是 ((k+1))-plex；
    
2. 因此最大 ((k-1))-plex 可作为最大 (k)-plex 的有效 lower bound；
    
3. 较小 (k) 下获得的解、局部种子和剪枝信息可以向较大 (k) 传播；
    
4. 这可以减少重复搜索。
    

可以改写为：

```latex
The key observation behind BottomUpMKP is the monotonicity of \(k\)-plexes with respect to \(k\): any \((k-1)\)-plex is also a valid \(k\)-plex. Therefore, the solution obtained for \(k-1\) can be directly reused as a feasible lower bound for \(k\). Based on this observation, BottomUpMKP solves the instances in increasing order of \(k\), progressively strengthening lower bounds and reusing previously discovered local solutions as search seeds.
```

这样读者会更容易理解为什么选择 bottom-up，而不仅仅是知道算法按 (k=2,3,\ldots,K) 遍历。

## 4. `P_{kc}\leftarrow P_{k-1}\backslash V_{G_k}` 这一行符号疑似错误

你现在写的是：

```latex
P_{kc}\leftarrow P_{k-1}\backslash V_{G_k}
```

从语义上看，你应该是想取 (P_{k-1}) 在当前 reduced graph (G_k) 中仍然保留的部分，即：

[  
P_{kc} \leftarrow P_{k-1} \cap V_{G_k}  
]

而不是：

[  
P_{k-1}\setminus V_{G_k}  
]

后者表示“从 (P_{k-1}) 中删除所有属于 (G_k) 的顶点”，通常会得到不在当前图中的残留顶点，语义相反。

建议改为：

```latex
$P_{kc}\leftarrow P_{k-1}\cap V_{G_k}$\;
```

并在正文中解释：

```latex
Since \(G_k\) may be reduced independently, some vertices in \(P_{k-1}\) may not survive in \(G_k\). We therefore reuse \(P_{k-1}\cap V_{G_k}\) as a feasible inherited solution candidate.
```

不过这里还要注意：  
(P_{k-1}\cap V_{G_k}) 虽然仍然是一个 (k)-plex，但它可能不是很好的 seed，因为删去顶点后规模可能下降。最好说明这是一个 inherited candidate，而不是直接等同于 (P_{k-1})。

## 5. “(\mathcal G_{k+1}\sqsubseteq \mathcal G_k)” 的方向需要核查

你写道：

```text
which guarantees that \(\mathcal G_{k+1}\sqsubseteq \mathcal G_k\)
```

这句话需要特别谨慎。对于最大 (k)-plex，随着 (k) 增大，约束通常变松，因此 reduced graph 一般不应更小，除非你的 `MultiPreMKP` 采用了某种特殊定义。

如果 (\sqsubseteq) 表示“子图关系”，那么你这句话意味着：

[  
\mathcal G_{k+1} \subseteq \mathcal G_k  
]

但直觉上更可能是：

[  
\mathcal G_k \subseteq \mathcal G_{k+1}  
]

因为 (k+1)-plex 的搜索空间通常包含 (k)-plex 的搜索空间。

建议你明确：

1. (\sqsubseteq) 的定义；
    
2. reduced graph 的包含方向；
    
3. 该性质是由哪条归约规则保证的；
    
4. 如果方向确实是 (\mathcal G_{k+1}\sqsubseteq \mathcal G_k)，需要解释为什么更大的 (k) 反而图更小。
    

否则审稿人很容易质疑这里的正确性。

## 6. “(|\mathcal P_{k+1}|\ge |\mathcal P_k|+2)” 过强，可能不成立

你写道：

```text
\(|\mathcal P_{k+1}|\ge |\mathcal P_k|+2\)
```

这个性质一般不能保证。由单调性只能得到：

[  
|\mathcal P_{k+1}| \ge |\mathcal P_k|  
]

不能保证至少增加 2。最大 (k)-plex 的大小可能在多个连续 (k) 上保持不变。

建议改为：

```latex
\(|\mathcal P_{k+1}|\ge |\mathcal P_k|\)
```

如果你的启发式确实会尝试构造至少大 2 的候选解，也应写成：

```latex
MultiPreMKP attempts to find increasingly larger heuristic solutions as \(k\) grows.
```

而不要写成 guarantee。

## 7. `UB_k = LB_{k-1}+\omega` 需要补充理论依据，否则建议弱化

算法中仍然保留：

```latex
$LB_k,UB_k\leftarrow \max(|P_k|,LB_{k-1}),LB_{k-1}+\omega$
```

如果正文说这是 “size-relationship-based bounding”，就需要给出 lemma 或 proposition 证明：

[  
\omega_k \le \omega_{k-1}+\omega  
]

否则这个上界在理论上站不稳。

如果暂时没有证明，建议不要把它称为 rigorous upper bound，而改称：

```text
a coarse global reference bound
```

或者直接从算法里移除 `UB_k`，因为当前后续伪代码并未真正使用它。  
如果保留，应在正文中说明它具体在哪里参与剪枝，例如：

```latex
If \(UB_k \le |P_k|\), the search for parameter \(k\) can be skipped.
```

否则 `UB_k` 会显得是一个没有实际用途的变量。

## 8. 2-hop 子图的正确性说明需要更精确

你写道：

```text
since the diameter of any k-plex is no greater than 2 by Theorem...
```

这句话可能过于绝对。通常需要满足 (k)-plex 的规模足够大，例如大小至少 (2k-1) 一类条件时，直径不超过 2。否则小规模 (k)-plex 的直径不一定受这个性质约束。

建议改为更谨慎的表述：

```latex
For any \(k\)-plex whose size is at least \(2k-1\), the diameter is at most two. Since BottomUpMKP only needs to find a solution larger than the current lower bound \(LB_k\), the two-hop restriction is safe when \(LB_k+1\ge 2k-1\). For the remaining cases, a fallback search without the two-hop restriction is required.
```

如果你算法没有 fallback，那么需要在算法前提中加入：

[  
LB_k+1\ge 2k-1  
]

否则这是一个潜在正确性漏洞。

## 10. BRB 的介绍目前偏口语化，需要学术化

## 12. `BRB_Bound` 的输出描述有不一致

算法写：

```latex
\KwOut{$ub$, the upper bound of $|S|$ in this branch}
```

但返回：

```latex
\Return{$C^*\leftarrow C_L\cup C_R, UB^*\leftarrow |S|+UB_L+UB_R$}
```

这里有两个问题：

1. 输出写的是 `ub`，返回的是 `C^*` 和 `UB^*`；
    
2. “upper bound of (|S|)” 不准确，因为 (|S|) 已经固定，真正要上界的是该分支可达到的最大解大小。
    

建议改为：

```latex
\KwOut{the reduced candidate set \(C^*\) and an upper bound \(UB^*\) on the largest \(k\)-plex reachable from \(S\cup C\)}
```

或者：

```latex
\KwOut{\(C^*\), the reduced candidate set, and \(UB^*\), an upper bound on \(|S\cup X|\) for any feasible \(X\subseteq C^*\)}
```

在 BottomUpMKP 中也对应写成：

```latex
$C,ub\leftarrow \text{BRB\_Bound}(g,k,S,C,P,v^*)$\;
```

而不是：

```latex
$ub,C\leftarrow ...$
```

保持顺序一致。

## 19. `V_1` 和 `V_2` 的定义可以简化

## 20. branching 规则的逻辑需要进一步解释，否则像“启发式拼接”

目前 branching 方法的描述说它是 kPlexT 的改进版，但没有解释每一步为什么合理。建议在正文中按三层逻辑解释：

1. **First priority:** choose vertices with minimum degree into (S)，因为它们更容易违反当前 partial solution 的可行性；
    
2. **Second priority:** among them, choose those with minimum degree in (S\cup C)，因为它们对整体可扩展性限制更强；
    
3. **Fallback pivoting:** choose a high-degree pivot (w)，然后在其 non-neighbors 中选择更可能造成冲突的点进行分支。
    

可以补充一段：

```latex
The first two stages prioritize vertices that are structurally tight with respect to the current partial solution. Such vertices are more likely to induce feasibility violations and hence lead to stronger pruning after branching. If no such vertex is available, the method falls back to a pivoting strategy: it selects a highly connected pivot \(w\) and branches on a critical non-neighbor of \(w\), following the intuition that any large \(k\)-plex containing \(w\) can tolerate only a limited number of non-neighbors of \(w\).
```

## 26. 建议增加一个“Correctness”小段

当前章节直接从算法描述进入 bounding 和 branching，但没有系统证明 correctness。建议增加一个短小节：

```latex
\subsection{Correctness}
```

可以证明三个点：

1. **Lower-bound inheritance:** (P_{k-1}) 是合法 (k)-plex；
    
2. **Anchored enumeration:** 删除式顶点枚举不会漏掉最优解；
    
3. **BRB soundness:** bounding 和 reduction 只删除不可能产生更优解的候选，branching 覆盖 include/exclude 两种情况，除非 `must_include` 有安全证明。
    

结构可以是：

```latex
Lemma 1. For every \(k\ge 2\), the inherited solution from \(k-1\) is a feasible \(k\)-plex.

Lemma 2. For a fixed \(k\), the anchored enumeration over the degeneracy ordering considers every feasible \(k\)-plex at least once.

Lemma 3. BRB preserves optimality under the proposed reduction and branching rules.

Theorem. BottomUpMKP returns a maximum \(k\)-plex for every \(2\le k\le K\).
```

这对论文非常重要。否则目前这一节更像“算法说明”，还不像“可发表算法”。

## 29. 建议在算法描述中区分 “heuristic solution” 和 “exact solution”

你现在的 (\mathcal P_k)、(P_k)、(\mathcal S_{v^*}) 三类解容易混淆：

```latex
We use \(\hat{P}_k\) to denote the heuristic solution returned by preprocessing, \(P_k\) to denote the incumbent solution maintained by exact search, and \(\mathcal S_v\) to denote the anchor-specific seed associated with vertex \(v\).
```

然后把 `\mathcal P_k` 改为 `\hat{P}_k`。  
这样可以显著降低读者理解成本。

## ✅30. 建议修改后的段落示例

下面给出一个更自然的开头段落示例，你可以参考其风格：

```latex
\section{Bottom-Up Multi-\(k\) Maximum \(k\)-Plex Search}

This section presents \textsc{BottomUpMKP}, a bottom-up framework for solving the multi-\(k\) maximum \(k\)-plex problem. Given a graph \(G=(V,E)\) and an integer \(K\ge 2\), the goal is to compute a set of solutions \(\{P_k\mid 2\le k\le K\}\), where each \(P_k\) is a maximum \(k\)-plex of \(G\).

The framework is based on the monotonicity of \(k\)-plexes: every \((k-1)\)-plex is also a feasible \(k\)-plex. Therefore, the incumbent solution obtained for \(k-1\) can be reused as a valid lower bound for \(k\). In addition, local solutions discovered during earlier searches are maintained as anchor-specific seeds and reused to accelerate subsequent searches. By solving the instances in increasing order of \(k\), \textsc{BottomUpMKP} progressively strengthens lower bounds and reduces redundant search across different relaxation levels.
```

这个版本比原文更能体现方法动机。

# 最核心的修改优先级

如果时间有限，我建议你优先修改以下 8 点：

1. 把所有 “maximum plex” 改为 “maximum (k)-plex”。
    
2. 修正 `P_{kc}\leftarrow P_{k-1}\backslash V_{G_k}` 为交集。
    
3. 检查并修正 (\mathcal G_{k+1}\sqsubseteq \mathcal G_k) 的包含方向。
    
4. 删除或弱化 (|\mathcal P_{k+1}|\ge |\mathcal P_k|+2)。
    
5. 补充 2-hop 子图安全条件，尤其是 (LB_k+1\ge 2k-1)。
    
6. 修正 `BRB_Bound` 和 `BRB_Branch` 的输入输出不一致。
    
7. 把口语表达如 “cherry-pick”“squishes”“gist” 改成学术表达。
    
8. 增加 correctness 小节，至少证明 anchored enumeration、lower-bound inheritance 和 BRB soundness。
    

完成这些修改后，这一节会从“算法草案说明”明显提升为“较规范的算法论文小节”。
